# Plan: Current Document Feature for Live View

## Context

Moderators need to project a specific attachment or motion document to all attendees during a
meeting. Each agenda point gets two new nullable FK columns (`current_attachment_id`,
`current_motion_id`) that are mutually exclusive — setting one clears the other. The moderator
controls appear on **both** the manage page (agenda-point-tools) and the moderate page tools card.
The live page always shows a preview pane (placeholder when nothing is set; iframe/link when a
document is active). The current doc persists per-AP and is not auto-cleared when the active AP
changes.

---

## Architecture

```
Moderator sets current doc
  → POST set-current route (moderate_access middleware)
  → DB: SetCurrentAttachment / SetCurrentMotion / ClearCurrentDocument
  → Publish "speakers-updated" SSE event
  → Return updated AttachmentListPartial / MotionListPartial (inline HTMX swap)

Attendee live page
  ← SSE "speakers-updated" fires
  ← AttendeeSpeakersListPartial re-rendered (includes OOB swap for #live-current-doc)
  ← Preview pane updated: iframe (PDF/image) or download link, or placeholder
```

## Step-by-Step Implementation

### Step 1 — DB Migration

**`023_current_doc_on_ap.up.sql`**:
```sql
ALTER TABLE agenda_points ADD COLUMN current_attachment_id INTEGER REFERENCES agenda_attachments(id);
ALTER TABLE agenda_points ADD COLUMN current_motion_id      INTEGER REFERENCES motions(id);
```

**`023_current_doc_on_ap.down.sql`**:
```sql
ALTER TABLE agenda_points DROP COLUMN current_attachment_id;
ALTER TABLE agenda_points DROP COLUMN current_motion_id;
```

### Step 2 — SQL Queries

Add to **`internal/repository/sqlite/queries/agenda.sql`**:
```sql
-- name: SetCurrentAttachment :exec
UPDATE agenda_points
   SET current_attachment_id = ?, current_motion_id = NULL
 WHERE id = ?;

-- name: SetCurrentMotion :exec
UPDATE agenda_points
   SET current_motion_id = ?, current_attachment_id = NULL
 WHERE id = ?;

-- name: ClearCurrentDocument :exec
UPDATE agenda_points
   SET current_attachment_id = NULL, current_motion_id = NULL
 WHERE id = ?;
```

Run `task generate:db` after.

### Step 3 — Model Update

**`internal/repository/model/agenda_point.go`** — add:
```go
CurrentAttachmentID *int64
CurrentMotionID     *int64
```

### Step 4 — Repository Interface

Add to **`internal/repository/repository.go`**:
```go
SetCurrentAttachment(ctx context.Context, agendaPointID, attachmentID int64) error
SetCurrentMotion(ctx context.Context, agendaPointID, motionID int64) error
ClearCurrentDocument(ctx context.Context, agendaPointID int64) error
```

### Step 5 — Attendee Blob Serve Route

The existing blob download requires `manage_access`. Add a new attendee-accessible endpoint.

Add to **`routes.yaml`**:
```yaml
- path: /committee/{slug}/meeting/{meeting_id}/current-doc
  methods:
    - verb: GET
      handler: ServeCurrentDocument
      middleware:
        - session
        - auth
        - committee_access
        - meeting_access
      raw: true
```

Handler **`ServeCurrentDocument`** (in `internal/handlers/current_doc.go`):
1. Parse `meetingID`
2. `GetMeetingByID` → check `meeting.CurrentAgendaPointID` (404 if none)
3. `GetAgendaPointByID` → check `ap.CurrentAttachmentID` or `ap.CurrentMotionID` (404 if neither)
4. Resolve blob: `GetAttachmentByID` / `GetMotionByID` → `GetBlobByID`
5. Read from `h.Storage`, stream with `Content-Type: <blob.ContentType>`,
   `Content-Disposition: inline; filename="<blob.Filename>"`

### Step 6 — Set / Clear Routes

Add to **`routes.yaml`** (using `moderate_access` — superset of `manage_access`, covers
chairpersons, chair attendees, and designated moderators):
```yaml
- path: /committee/{slug}/meeting/{meeting_id}/agenda-point/{agenda_point_id}/attachment/{attachment_id}/set-current
  methods:
    - verb: POST
      handler: SetCurrentAttachment
      middleware: [session, auth, committee_access, meeting_access, moderate_access]
      template:
        package: github.com/Y4shin/conference-tool/internal/templates
        type: AttachmentListPartial
        input_type: AttachmentListPartialInput

- path: /committee/{slug}/meeting/{meeting_id}/agenda-point/{agenda_point_id}/motion/{motion_id}/set-current
  methods:
    - verb: POST
      handler: SetCurrentMotion
      middleware: [session, auth, committee_access, meeting_access, moderate_access]
      template:
        package: github.com/Y4shin/conference-tool/internal/templates
        type: MotionListPartial
        input_type: MotionListPartialInput

- path: /committee/{slug}/meeting/{meeting_id}/agenda-point/{agenda_point_id}/clear-current
  methods:
    - verb: POST
      handler: ClearCurrentDocument
      middleware: [session, auth, committee_access, meeting_access, moderate_access]
      template:
        package: github.com/Y4shin/conference-tool/internal/templates
        type: AttachmentListPartial
        input_type: AttachmentListPartialInput
```

Each handler in `internal/handlers/current_doc.go`:
1. Parse `agendaPointID` (and `attachmentID` / `motionID` where applicable)
2. Call `repo.SetCurrentAttachment` / `repo.SetCurrentMotion` / `repo.ClearCurrentDocument`
3. Call `h.publishCurrentDocumentChanged(meetingID)`
4. Load and return updated partial via `h.loadAttachmentListPartial` or `h.loadMotionListPartial`

### Step 7 — `publishCurrentDocumentChanged`

Add to `internal/handlers/live.go`:
```go
func (h *Handler) publishCurrentDocumentChanged(meetingID int64) {
    mid := meetingID
    h.Broker.Publish(broker.SSEEvent{Event: "speakers-updated", Data: []byte("{}"), MeetingID: &mid})
}
```

Reuses "speakers-updated" so the live page SSE stream picks it up without new event types or
routes. The moderate/manage streams also listen to "speakers-updated" but the resulting reload
(speakers list + attendees) is lightweight and acceptable.

### Step 12 — New Template: `live_current_doc.templ`

```go
type LiveCurrentDocInput struct {
    CommitteeSlug   string
    MeetingIDString string
    Doc             *LiveCurrentDocInfo  // nil = no current doc
}

// DocURL builds /committee/{slug}/meeting/{id}/current-doc?v={blobID}
// The ?v param is a cache-buster so the browser reloads the iframe when the doc changes.
func (i *LiveCurrentDocInput) DocURL(ctx context.Context) templ.SafeURL
```

Helper (unexported Go function, not a template function):
```go
func isPreviewable(contentType string) bool {
    switch contentType {
    case "application/pdf", "image/jpeg", "image/png", "image/gif", "image/webp", "image/svg+xml":
        return true
    }
    return false
}
```

```templ
templ LiveCurrentDocPartial(input LiveCurrentDocInput) {
    <div id="live-current-doc">
        if input.Doc == nil {
            <p class="live-doc-placeholder">No document currently displayed.</p>
        } else if isPreviewable(input.Doc.ContentType) {
            <iframe src={ input.DocURL(ctx) } class="live-doc-iframe" title={ input.Doc.Filename }></iframe>
        } else {
            <p>
                @IconLabelLink(input.DocURL(ctx), IconDownload, input.Doc.Filename, "left")
                <span>(download to view)</span>
            </p>
        }
    </div>
}
```

### Step 13 — Live Page Template Layout Changes

**`internal/templates/meeting_live.templ`**:

Add `CurrentDoc *LiveCurrentDocInfo` to `MeetingLiveInput`.

**Desktop** — the left agenda panel (`LiveAgendaPanel`) becomes a two-part vertical stack:

The `#live-current-doc` element always exists in the DOM (placeholder or iframe), so the OOB swap
from the SSE partial always has a valid target.

**Mobile** — add a document icon button to the agenda card header (left of the existing full-agenda

```templ

        class={ templ.KV("live-doc-btn--hidden", input.CurrentDoc == nil) }

    [document icon]

```

Add a `<dialog id="live-doc-modal">` (same pattern as existing agenda/history modals) containing

For the mobile button visibility to update via SSE, add a second small OOB swap element in
`AttendeeSpeakersListPartial` that re-renders only the button:
```templ

```
