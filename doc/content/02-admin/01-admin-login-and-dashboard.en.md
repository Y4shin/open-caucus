# Admin Login and Dashboard

Use this page to open the admin area and confirm you are ready for admin tasks.

## Who this is for

Users with an admin account who manage committees, accounts, and platform policies.

## Before you start

1. Confirm you have admin credentials.
2. Make sure another non-admin account is not still signed in in the same browser.
3. If password login is turned off in your setup, confirm which admin login method you should use.

## Step-by-step

1. Open the admin login page.
2. Sign in with the available method:
   - Password login: enter username/password and click **Login**.
   - Single sign-on login: click **Login with OAuth**.
3. Confirm you are redirected to the admin dashboard.
4. Review dashboard summaries (committees and accounts) to check everything looks correct.
5. Use dashboard actions for the next step:
   - Click **Manage Accounts** to open account management.
   - Click **Assign Accounts** in a committee row to manage memberships.
   - Use **Create Committee** to add a new committee.

## What you should see

- Successful admin login takes you to the dashboard, not to a normal user page.
- Dashboard shows committee management plus direct links to account and membership tasks.
- Admin-only controls are visible only in this area.

## If something goes wrong

- Login is rejected:
  Re-check username/password and try again.
- You land in a non-admin area:
  Sign out and log in with an admin account.
- You receive access denied when opening admin pages:
  The signed-in account does not have admin permission.
- Password login is unavailable:
  Password login may be turned off in your setup; use the admin login method your team configured.

## What happens next

Continue with [Accounts and Committee Management](/docs/02-admin/02-accounts-and-committee-management) for create/update operations.
