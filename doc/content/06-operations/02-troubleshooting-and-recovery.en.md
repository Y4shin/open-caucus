# Troubleshooting and Recovery

Use this page for common live-operation issues and fast recovery steps.

## Who this is for

Admins and chairpersons supporting live meetings.

## Before you start

1. Confirm the exact symptom (what the user sees, and on which page).
2. Confirm user role and meeting context.
3. Apply the smallest safe fix first.

## Common incidents and fixes

## 1) Guest lost access code

1. Open the attendee list in moderation.
2. Use **Recovery link** for that guest.
3. Share the recovery link or recovery QR directly with the same person.
4. Confirm they return to the live page.

## 2) Wrong active meeting selected

1. Open the committee dashboard.
2. Check which meeting is marked active.
3. Switch active status to the correct meeting.
4. Ask users to re-open the meeting page.

## 3) UI looks stale or inconsistent

1. Use the local **Refresh** control in the affected panel.
2. If needed, reload the page once.
3. If session context changed, log out and log in again.

## 4) Vote stuck in counting state

1. Re-open the vote panel in **Tools**.
2. Complete remaining counting entries.
3. Close the vote again.
4. Archive only after final results are confirmed.

## 5) Guest cannot join

1. Check whether guest signup is open.
2. Re-share join QR/link if needed.
3. Confirm meeting secret entry is correct.

## If something goes wrong

- The first fix did not work:
  Re-check role, committee, and meeting context before deeper changes.
- Multiple users report different symptoms:
  Stabilize the active meeting and signup state first, then retest.
- Repeated session confusion:
  Ask affected users to log out and re-enter with the correct flow.
