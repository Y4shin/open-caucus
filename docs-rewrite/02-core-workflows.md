# Core Workflows

This file defines the main user journeys that docs should cover as step-by-step procedures.

## 1) Join and Participate in a Meeting (Member/Attendee)

1. Open committee page.
2. Join active meeting.
3. Complete join/login flow.
4. Use live page (speaker queue, votes, current document).

Key docs:

- `04-members-attendees/01-member-join-flow.en.md`
- `04-members-attendees/02-guest-signup-and-attendee-login.en.md`
- `04-members-attendees/03-live-page-self-service-and-voting.en.md`

## 2) Run a Meeting (Chairperson)

1. Start/close meeting session from committee dashboard.
2. Open Moderate workspace.
3. Manage attendees and speakers.
4. Handle agenda points and documents.
5. Run votes and review outcomes.

Key docs:

- `03-chairperson/01-committee-dashboard-and-meeting-lifecycle.en.md`
- `03-chairperson/02-moderate-workspace-overview.en.md`
- `03-chairperson/05-speakers-moderator-and-quotation.en.md`
- `03-chairperson/07-vote-moderation-open-and-secret.en.md`

## 3) Build and Adjust Agenda (Chairperson)

1. Create/edit agenda points manually.
2. Import agenda text and review diff.
3. Apply import and verify results.

Key docs:

- `03-chairperson/03-agenda-management-and-import.en.md`

## 4) Manage Meeting Documents (Chairperson/Attendees)

1. Upload attachments to agenda point.
2. Set/clear current document.
3. Verify attendee live view reflects current doc.

Key docs:

- `03-chairperson/06-attachments-and-current-document.en.md`

## 5) Verify Votes Publicly (Any User)

1. Open receipts vault.
2. Refresh/load locally stored receipts.
3. Verify open/secret receipts.

Key docs:

- `05-public-verification/01-receipts-vault-and-receipt-verification.en.md`
